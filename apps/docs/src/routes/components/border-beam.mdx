# BorderBeam

Decorative component that renders a moving beam along a container border.

## When To Use

- Use when a container needs stronger visual emphasis without introducing business state semantics.
- Suitable for login panels, recommendation cards, AI modules, and key CTA blocks.
- As a decorative effect, it should not replace focus rings, validation borders, or status feedback.

## API

Common props ref: Common props. Solid uses `class` and `classList` instead of React `className`.

The child must resolve to a single `HTMLElement`; text and SVG children are rendered without the beam.
The resolved host element should provide a positioning context, for example `position: relative`.

| Property   | Description                                                                                                        | Type                                             | Default | Version |
| ---------- | ------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------ | ------- | ------- |
| `children` | Decorated content.                                                                                                 | `JSX.Element`                                    | `-`     | `6.4.0` |
| `color`    | Beam color configuration. `percent` uses the `0 ~ 100` input range and BorderBeam reserves transparent tail space. | `string \| { color: string; percent: number }[]` | `-`     | `6.4.0` |
| `outset`   | Outset distance of the beam layer from the container edge. Set to `0` for clipped containers.                      | `number \| string`                               | `-`     | `6.4.0` |

## Solid Common Props

| Property    | Description                                                | Type                                   | Default |
| ----------- | ---------------------------------------------------------- | -------------------------------------- | ------- |
| `prefixCls` | Custom CSS class prefix.                                   | `string`                               | `-`     |
| `class`     | Additional class applied to the generated beam element.    | `string`                               | `-`     |
| `classList` | Conditional classes applied to the generated beam element. | `Record<string, boolean \| undefined>` | `-`     |
| `style`     | Inline style merged into the generated beam element.       | `JSX.CSSProperties \| string`          | `-`     |

## Design Token

BorderBeam has no component-specific token. It follows global tokens, including `colorPrimary`,
`colorPrimaryHover`, and `lineWidth`. `theme.components.BorderBeam.lineWidth` can override the beam
line width for this component.

## FAQ

### How does BorderBeam behave when reduced motion is enabled?

`BorderBeam` treats the beam as a decorative effect. When `prefers-reduced-motion: reduce` is active,
the beam effect is hidden.

### What does `percent` mean in `color`?

`percent` represents the authored stop position and accepts values from `0` to `100`. BorderBeam maps
those stops into the visible beam segment and reserves the trailing area for transparent fade-out so
the moving tail stays visible.

### Why is `BorderBeam` not working?

`BorderBeam` needs to resolve the actual DOM node from `children` and insert the beam layer into that
node. Make sure the wrapped content is a native DOM element or a component that resolves to a DOM
element. Otherwise BorderBeam cannot locate the real container and the beam cannot be rendered.

The beam layer is positioned with `position: absolute`, so the resolved DOM node also needs to provide
a positioning context. In most cases, set `position: relative` on the wrapped element.
