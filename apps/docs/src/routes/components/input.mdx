## API

### Input

| Property        | Description                                                                                                         | Type                                                                   | Default    |
| --------------- | ------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------- | ---------- |
| `size`          | Input size. Defaults to the current ConfigProvider component size.                                                  | `'small' \| 'middle' \| 'large'`                                       | `-`        |
| `status`        | Validation visual status.                                                                                           | `'error' \| 'warning'`                                                 | `-`        |
| `variant`       | Visual variant of the input wrapper.                                                                                | `'outlined' \| 'borderless' \| 'filled' \| 'underlined'`               | `outlined` |
| `bordered`      | Compatibility alias. `false` maps to `variant="borderless"` when `variant` is not set.                              | `boolean`                                                              | `true`     |
| `addonBefore`   | Compatibility addon rendered before the input. Prefer `Space.Compact` for new layouts.                              | `JSX.Element`                                                          | `-`        |
| `addonAfter`    | Compatibility addon rendered after the input. Prefer `Space.Compact` for new layouts.                               | `JSX.Element`                                                          | `-`        |
| `prefix`        | Element rendered before the input value.                                                                            | `JSX.Element`                                                          | `-`        |
| `suffix`        | Element rendered after the input value.                                                                             | `JSX.Element`                                                          | `-`        |
| `allowClear`    | Shows a clear button when the input has a value. Object form customizes the icon or disables the clear affordance.  | `boolean \| { clearIcon?: JSX.Element; disabled?: boolean }`           | `false`    |
| `showCount`     | Shows the current character count. Object form customizes count rendering.                                          | `boolean \| { formatter?: (info: CountFormatterInfo) => JSX.Element }` | `false`    |
| `count`         | Character count configuration for custom max, count strategy, display and exceed formatting.                        | `CountConfig`                                                          | `-`        |
| `classNames`    | Semantic class names for root, input, prefix, suffix, clear and count nodes. `wrapper` remains as a legacy alias.   | `InputSemanticClassNames \| ((info) => InputSemanticClassNames)`       | `-`        |
| `styles`        | Semantic inline styles for root, input, prefix, suffix, clear and count nodes. `wrapper` remains as a legacy alias. | `InputSemanticStyles \| ((info) => InputSemanticStyles)`               | `-`        |
| `rootClassName` | Compatibility class applied to the root wrapper. Use `class` for regular Solid styling.                             | `string`                                                               | `-`        |
| `prefixCls`     | Custom CSS class prefix for advanced compatibility.                                                                 | `string`                                                               | `-`        |
| `htmlSize`      | Native input `size` attribute.                                                                                      | `number`                                                               | `-`        |
| `value`         | Controlled input value.                                                                                             | `string \| number`                                                     | `-`        |
| `defaultValue`  | Initial uncontrolled input value.                                                                                   | `string \| number`                                                     | `-`        |
| `onChange`      | Callback when the input value changes.                                                                              | `(event: Event) => void`                                               | `-`        |
| `onPressEnter`  | Callback when Enter is pressed.                                                                                     | `(event: KeyboardEvent) => void`                                       | `-`        |
| `onClear`       | Callback when the clear button is clicked.                                                                          | `() => void`                                                           | `-`        |

### Input.Group

Deprecated compatibility wrapper. Prefer `Space.Compact` for new code.

| Property    | Description                   | Type                             | Default |
| ----------- | ----------------------------- | -------------------------------- | ------- |
| `compact`   | Enables compact group layout. | `boolean`                        | `false` |
| `size`      | Group size.                   | `'small' \| 'middle' \| 'large'` | `-`     |
| `prefixCls` | Custom CSS class prefix.      | `string`                         | `-`     |

### CountConfig

| Property          | Description                                                                                                                  | Type                                                     | Default |
| ----------------- | ---------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------- | ------- |
| `max`             | Maximum character count for warning state. Unlike native maxLength, this marks exceed state instead of truncating by itself. | `number`                                                 | `-`     |
| `strategy`        | Custom count strategy, useful for emoji or domain-specific length rules.                                                     | `(value: string) => number`                              | `-`     |
| `show`            | Whether to show count, or a render function for custom count content.                                                        | `boolean \| ((info: CountFormatterInfo) => JSX.Element)` | `-`     |
| `exceedFormatter` | Custom value formatter when count exceeds max.                                                                               | `(value: string, config: { max: number }) => string`     | `-`     |

### Input.TextArea

| Property        | Description                                                                | Type                                                                   | Default    |
| --------------- | -------------------------------------------------------------------------- | ---------------------------------------------------------------------- | ---------- |
| `autoSize`      | Autosize textarea rows. Object form supports minRows and maxRows.          | `boolean \| { minRows?: number; maxRows?: number }`                    | `false`    |
| `size`          | TextArea size. Defaults to the current ConfigProvider component size.      | `'small' \| 'middle' \| 'large'`                                       | `-`        |
| `allowClear`    | Shows a clear button when the textarea has a value.                        | `boolean \| { clearIcon?: JSX.Element; disabled?: boolean }`           | `false`    |
| `showCount`     | Shows the current character count. Object form customizes count rendering. | `boolean \| { formatter?: (info: CountFormatterInfo) => JSX.Element }` | `false`    |
| `count`         | Character count configuration shared with Input.                           | `CountConfig`                                                          | `-`        |
| `variant`       | Visual variant of the textarea.                                            | `'outlined' \| 'borderless' \| 'filled' \| 'underlined'`               | `outlined` |
| `status`        | Validation visual status.                                                  | `'error' \| 'warning'`                                                 | `-`        |
| `classNames`    | Semantic class names for root, textarea, clear and count nodes.            | `TextAreaSemanticClassNames \| ((info) => TextAreaSemanticClassNames)` | `-`        |
| `styles`        | Semantic inline styles for root, textarea, clear and count nodes.          | `TextAreaSemanticStyles \| ((info) => TextAreaSemanticStyles)`         | `-`        |
| `rootClassName` | Compatibility class applied to the root wrapper.                           | `string`                                                               | `-`        |
| `prefixCls`     | Custom CSS class prefix for advanced compatibility.                        | `string`                                                               | `-`        |
| `value`         | Controlled textarea value.                                                 | `string \| number`                                                     | `-`        |
| `defaultValue`  | Initial uncontrolled textarea value.                                       | `string \| number`                                                     | `-`        |
| `onPressEnter`  | Callback when Enter is pressed.                                            | `(event: KeyboardEvent) => void`                                       | `-`        |
| `onResize`      | Callback when the textarea resize event fires.                             | `(size: { width: number; height: number }) => void`                    | `-`        |
| `onClear`       | Callback when the clear button is clicked.                                 | `() => void`                                                           | `-`        |

### Input.Search

Supports all Input props plus the following props.

| Property         | Description                                                                                                     | Type                                                               | Default |
| ---------------- | --------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------ | ------- |
| `enterButton`    | Render a search button after the input. True renders the default button text; string or JSX customizes content. | `boolean \| string \| JSX.Element`                                 | `false` |
| `loading`        | Shows loading state on the search icon or enter button.                                                         | `boolean`                                                          | `false` |
| `searchIcon`     | Custom search icon when enterButton is not used.                                                                | `JSX.Element`                                                      | `-`     |
| `inputPrefixCls` | Custom CSS class prefix for the internal input.                                                                 | `string`                                                           | `-`     |
| `classNames`     | Semantic class names including Input nodes and `button`.                                                        | `SearchSemanticClassNames \| ((info) => SearchSemanticClassNames)` | `-`     |
| `styles`         | Semantic inline styles including Input nodes and `button`.                                                      | `SearchSemanticStyles \| ((info) => SearchSemanticStyles)`         | `-`     |
| `onSearch`       | Callback when clicking search, pressing Enter, or clearing the input.                                           | `(value, event?, info?: { source?: "input" \| "clear" }) => void`  | `-`     |

### Input.Password

Supports most Input props plus password visibility controls.

| Property           | Description                                                              | Type                                                                             | Default |
| ------------------ | ------------------------------------------------------------------------ | -------------------------------------------------------------------------------- | ------- |
| `action`           | Trigger action for toggling password visibility.                         | `'click' \| 'hover'`                                                             | `click` |
| `suffix`           | Extra suffix content rendered before the visibility toggle.              | `JSX.Element`                                                                    | `-`     |
| `inputPrefixCls`   | Custom CSS class prefix for the internal input.                          | `string`                                                                         | `-`     |
| `iconRender`       | Custom password visibility toggle icon.                                  | `(visible: boolean) => JSX.Element`                                              | `-`     |
| `visibilityToggle` | Whether to show the toggle icon, or controlled visibility configuration. | `boolean \| { visible?: boolean; onVisibleChange?: (visible: boolean) => void }` | `true`  |

### Input.OTP

| Property        | Description                                                     | Type                                                         | Default    |
| --------------- | --------------------------------------------------------------- | ------------------------------------------------------------ | ---------- |
| `length`        | Number of OTP input boxes.                                      | `number`                                                     | `6`        |
| `mask`          | Masks displayed characters without changing the original value. | `boolean \| string`                                          | `false`    |
| `formatter`     | Formats each displayed character.                               | `(value: string) => string`                                  | `-`        |
| `separator`     | Renders a separator after the input at the specified index.     | `JSX.Element \| ((index: number) => JSX.Element)`            | `-`        |
| `autoComplete`  | Autocomplete attribute for OTP autofill, such as one-time-code. | `string`                                                     | `-`        |
| `type`          | Native input type for each OTP field.                           | `string`                                                     | `-`        |
| `value`         | Controlled OTP value.                                           | `string`                                                     | `-`        |
| `defaultValue`  | Initial uncontrolled OTP value.                                 | `string`                                                     | `-`        |
| `size`          | Size of each OTP input.                                         | `'small' \| 'middle' \| 'large'`                             | `middle`   |
| `status`        | Validation visual status.                                       | `'error' \| 'warning'`                                       | `-`        |
| `variant`       | Visual variant of OTP inputs.                                   | `'outlined' \| 'borderless' \| 'filled' \| 'underlined'`     | `outlined` |
| `classNames`    | Semantic class names for root, input and separator nodes.       | `OTPSemanticClassNames \| ((info) => OTPSemanticClassNames)` | `-`        |
| `styles`        | Semantic inline styles for root, input and separator nodes.     | `OTPSemanticStyles \| ((info) => OTPSemanticStyles)`         | `-`        |
| `rootClassName` | Compatibility class applied to the root wrapper.                | `string`                                                     | `-`        |
| `prefixCls`     | Custom CSS class prefix for advanced compatibility.             | `string`                                                     | `-`        |
| `onInput`       | Callback on every OTP input value update.                       | `(value: string[]) => void`                                  | `-`        |
| `onChange`      | Callback when all OTP fields are filled.                        | `(value: string) => void`                                    | `-`        |

### Ref

| Component                                   | Ref API                                                                                                |
| ------------------------------------------- | ------------------------------------------------------------------------------------------------------ |
| `Input` / `Input.Search` / `Input.Password` | `{ focus(options), blur(), select(), setSelectionRange(start, end, direction), input, nativeElement }` |
| `Input.TextArea`                            | `{ focus(options), blur(), resizableTextArea, nativeElement }`                                         |
| `Input.OTP`                                 | `{ focus(), blur(), nativeElement }`                                                                   |

`focus` supports `{ preventScroll?: boolean; cursor?: 'start' | 'end' | 'all' }` for `Input` and `TextArea`.

### Semantic DOM

| Component        | Keys                                                            |
| ---------------- | --------------------------------------------------------------- |
| `Input`          | `root`, `input`, `prefix`, `suffix`, `clear`, `count`           |
| `Input.Search`   | `root`, `input`, `prefix`, `suffix`, `clear`, `count`, `button` |
| `Input.TextArea` | `root`, `textarea`, `clear`, `count`                            |

`wrapper` is still accepted as a legacy alias for `root`.
