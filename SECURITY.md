# Security Policy

## Reporting a Vulnerability

Please do not report security vulnerabilities in public issues.

Use GitHub's private vulnerability reporting or GitHub Security Advisories for this repository. Include:

- Affected package and version.
- Reproduction steps or proof of concept.
- Expected impact.
- Any known mitigation.

We will review reports as soon as practical and coordinate fixes before public disclosure.

## Supported Versions

Until the first stable release, security fixes target the latest code on `main` and the latest published package versions.
